The Effects of Mode Choice Model Consistency between Activity-based Models and Microsimulation Tools

3 Findings

To effectively compare the differences between the five scenarios, we calculated how the mode choices were distributed across the population.

Figure 3.1 shows the total modal share of the population. We hypothesized that the ActivitySim All model (2.5) would produce the most accurate mode choice results. Surprisingly, however, Figure 3.1 shows that this was not the case. Instead, the Beam Default MNL model (2.1) produced the closest approximation of total modal share.

Figure 3.1: The total modal split of each scenario compared with real world data.

Figure 3.2 displays how modal share is affected by vehicle ownership. Again, we were incorrect in our hypothesis that the ActivitySim All model (2.5) would align closest to the target shares. The Person model (2.3) best predicted the auto deficient values, the Path model (2.2) best predicted the auto sufficient values, and the BEAM Default MNL model (2.1) best predicted the no auto values. In general though, none of the models gave an accurate representation of this modal share.

Figure 3.2: The modal split by vehicle ownership of each scenario compared with real world data.

Figure 3.1 and Figure 3.2 also show that different variable types in the utility equation yield different mode choice results. The Path (2.2), Person (2.3), and Location (2.4) attributes all affected the mode choice in different ways.

Figure 3.3 displays how mode choice for each scenario is affected by tour purpose. School, university, and work purpose trips varied the most among different scenarios.

Figure 3.3: The modal split by tour purpose of each scenario.

Overall, a microsimulation tool with a mode choice structure that mimics that of its parenting activity-based model does not seem to simulate mode choices accurately. However, we suggest that analysts attempt to rerun the scenarios that we have presented. Unfortunately, we did not calibrate the alternative specific constants for the ActivitySim scenarios. Rerunning the ActivitySim scenarios, but with calibrated constants, could yield more enticing results.
